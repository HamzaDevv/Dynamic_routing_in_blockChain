PCER-T v4 — What Changed from v3?

v4 takes the three strongest capabilities from RPL-MRHOF+Trust and grafts them surgically into PCER-T's QoS framework. The result is a hybrid that retains full QoS tag-awareness while gaining RPL's proven link-quality intelligence:

MRHOF ETX Link Metric Adaptive Hysteresis Dual Trust Compound Fractional QoS Tags Cubic Energy Floor Load-Aware Tie-Break
Imports: RFC 6552 MRHOF ETX | RPL MIN_IMPROVEMENT | RSOF NFR blacklist | BLET-IoT energy floor | T-OF load balance

The 3 RPL-MRHOF Ingredients Borrowed

① MRHOF ETX
RPL's Expected Transmission Count estimates how many attempts a link needs. v4 normalises this as a soft link-quality cost added to every hop evaluation — even Critical packets avoid flaky links if ETX is very poor.
② Hysteresis Band
RPL's MIN_IMPROVEMENT: a new route must beat the current preferred parent by ≥15% before PCER-T switches. This prevents oscillation when two hops alternate between near-equal costs under link noise.
③ Dual Trust Compound
RPL-MRHOF+Trust compounds ETX and trust. v4 does the same: a node's ETX penalty is amplified by (1−trust), so a low-trust node also pays a higher link-quality price — double punishment for chronic droppers.
v3 Retained
All v3 innovations kept: fractional QoS weights, cubic energy demotion at 8%, hard block at 3%, soft ETX penalty for trust 0.4–0.7, load-balancing tie-break by forwarding history.

PCER-T v4 Cost Function

// ══ HARD GUARDS (→ ∞) ══
IF nfr_blacklisted(v) AND v ≠ DST → ∞
IF battery(v) < 3% AND v ≠ DST → ∞

// ══ MRHOF ETX (Borrowed from RPL) ══
etx(v) = simulated_link_quality(u,v) // 1.0=perfect
etx_penalty = (etx(v) − 1.0) × 12 // 0 if perfect

// ══ DUAL TRUST COMPOUND (RPL × PCER-T) ══
tr = trust(v) // 0..1
trust_base = (1 − tr) × 20 // base penalty
etx_x_trust = etx_penalty × (1 − tr) // compound!

// ══ CUBIC ENERGY FLOOR (v3 retained) ══
bat = battery(v)
IF bat < 8% AND v ≠ DST:
e_penalty = (0.08/bat)³ × 100 // cubic
ELSE:
e_penalty = (1/bat) × 2 // normal

// ══ QoS TAG-ADAPTIVE WEIGHTS ══
w = {CRIT:[wd=0.70,we=0.08,wt=0.14,wetx=0.06,wl=0.02]
STD :[wd=0.38,we=0.30,wt=0.18,wetx=0.08,wl=0.06]
BULK:[wd=0.04,we=0.60,wt=0.20,wetx=0.10,wl=0.06]}

// ══ FINAL COST ══
delay_cost = (edge_ms + h_dist) × w.wd
energy_cost = e_penalty × w.we
trust_cost = trust_base × w.wt
etx_cost = (etx_penalty + etx_x_trust) × w.wetx
lb_cost = load_count(v) × 0.7 × w.wl

raw_cost = delay+energy+trust+etx+lb costs

// ══ HYSTERESIS GUARD (Borrowed from RPL) ══
IF prev_best exists AND raw_cost < prev_best:
IF improvement < 15% → keep prev_best
ELSE → switch to new, log MRHOF-switch

FINAL COST = raw_cost (after hysteresis)

Limitations & Future Work

PCER-T v4 still omits: full DIO trickle timer management, TSCH slot scheduling, cryptographic node authentication, and topology-level repair when DAG roots fail. ETX in this simulation is probabilistic, not measured from real radio. These remain open research directions for constrained 6LoWPAN deployments.
